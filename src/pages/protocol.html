Protocol

Identity that mirrors conscience.

Open People is an open, non-profit protocol for human-aligned digital identity. It makes intention observable, action verifiable, and consent portable.

Read the white paper View on GitHub

Principles

  • Consent before capability.
  • Transparency by default.
  • Proof over promise.
  • Forkability and portability.

The four-layer stack

Layer 1

Codex — Conscience

Defines the ethics and consent contracts that every action must honour. Encodes intention, red lines, and transparency rules as verifiable structure.

  • Alignment contracts
  • Consent schemas
  • Transparency ledger
Layer 2

Cortex — Execution

Translates declared intent into action. Plans, executes, and records outcomes with safety checks and audit trails embedded.

  • Ethical planners
  • Action attestations
  • Feedback observability
Layer 3

Assistant — Reflection

Maintains coherence by observing behaviour, catching drift, and guiding ethical course correction in real-time.

  • Consent stewards
  • Dual-channel explanations
  • Reflection prompts
Layer 4

Temporal — Memory

Preserves continuity without surveillance. Stores proofs, anchors history, and enables selective forgetting with portability.

  • Versioned identity states
  • Cryptographic anchors
  • Audit exports

Alignment signals

Alignment is observable. The protocol ships with telemetry that measures how closely actions follow declared conscience.

  • VR — Variance Reduction: matching reality to stated ethics.
  • DE — Decision Efficiency: speed without skipping safeguards.
  • QAA — Quality-Adjusted Acceptance: trusted outcomes per cycle.
  • FL — Feedback Latency: time to detect and correct drift.
  • CRI — Cultural Resolution Index: fit to local norms.

Operational loop

  1. Intake signal → unified event bus.
  2. Plan minimal, reversible actions.
  3. Verify against Codex red lines.
  4. Execute + observe outcomes.
  5. Report, learn, and anchor memory.

Every step is logged with proofs and available for audit.

APIs

The protocol exposes simple, composable endpoints. Everything returns proofs so actions remain portable.

POST /cortex/plan   → { plan, rationale, traceId }
POST /cortex/act    → { result | error }
POST /assistant/consent → { state, timestamp }
GET  /temporal/audit?trace=… → { records[], proofs }
